### R2-8 Fix Detail

**Bug:** In `hvac_zones.py restore_state_snapshot()`, the line `zone.zone_persons = state.get("zone_persons", zone.zone_persons)` overwrote the zone_persons that were just read from the config entry during `async_discover_zones()` with stale data from the snapshot. On restart:

1. `async_discover_zones()` reads fresh `zone_persons` from config entry ✓
2. `restore_state_snapshot()` overwrites with old snapshot value ✗

This means if the user changed zone persons via config flow, the change wouldn't take effect until the snapshot expired (4h) or was overwritten.

**Fix:** Removed the `zone.zone_persons` restore line. Config entry is the source of truth for person assignments. The `__person_zone_map` top-level key in the snapshot serves as the DB fallback if zone configs are empty.
