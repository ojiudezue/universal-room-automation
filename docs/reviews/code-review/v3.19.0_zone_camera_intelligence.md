### Pattern Compliance Notes

The v3.19.0 implementation follows several established URA patterns correctly:

1. **Additive-only integration**: Face recognition is purely additive to the existing camera detection pipeline. All failures return gracefully. The existing `_handle_camera_change` flow is preserved exactly.

2. **Config entry merge pattern**: `{**data, **options}` used correctly for both the integration-level face_recognition_enabled toggle (presence.py:568) and the zone-level camera config (config_flow.py:3833).

3. **Deep-copy zone update pattern**: Config flow uses `{k: dict(v) for k, v in ...}` (config_flow.py:3835), preventing the shallow copy bug from v3.8.2.

4. **Datetime serialization**: All datetime values use `.isoformat()` in state attributes, matching the pattern established after the v3.18.6 R1-1 fix.

5. **@callback decoration**: Methods called by HA's event system are properly decorated. Helper methods called from within @callback contexts correctly omit the decorator.

6. **Try/except isolation**: HVAC cross-coordinator access is wrapped in its own try/except (lines 1232-1243), preventing failures in the non-critical HVAC counter from blocking the critical signal dispatch.
